# Migrations

Міграції зберігаємо як послідовні SQL-файли.

## Принцип
- 1 міграція = 1 SQL-файл.
- Назва: `YYYYMMDDHHMMSS_slug.sql` (UTC або локальний час, головне — зростання).
- Міграції застосовуються послідовно за іменем файлу.
- Нові зміни — тільки через новий файл (не редагувати старі).

## Приклад
```
20260204183000_create_core_tables.sql
20260205110000_add_indexes.sql
```

## Як створювати
1. Створи новий файл у `supabase/migrations/`.
2. Додай SQL для зміни (create/alter/drop).
3. Проганяй в Supabase SQL Editor по черзі або через CLI.
